## Auto-Save Contract

**Debounce:** 500ms per NOTE-08 requirement

**Implementation:**
```typescript
function useDebounce<T>(value: T, delay: number): T {
  const [debouncedValue, setDebouncedValue] = useState<T>(value)

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value)
    }, delay)

    return () => clearTimeout(handler)
  }, [value, delay])

  return debouncedValue
}
```

**Auto-save trigger:**
- Only save when `debouncedContent !== initialContent`
- Skip save on initial render
- Show "Saving..." indicator during save operation

## Full-Text Search Contract

**Search vector update trigger:**
```sql
CREATE OR REPLACE FUNCTION notes_search_vector_update() RETURNS TRIGGER AS $$
BEGIN
  NEW.search_vector :=
    setweight(to_tsvector('english', coalesce(NEW.title, '')), 'A') ||
    setweight(to_tsvector('english', coalesce(NEW.content::text, '')), 'B');
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;
```

**Search query:**
- Title weight: 'A' (higher priority)
- Content weight: 'B' (lower priority)
- Language: 'english' for better stemming
- Ranking: `ts_rank()` ordered DESC

**Search operators:**
- `&` - AND (default)
- `|` - OR
- `!` - NOT
- `<->` - FOLLOWED BY (phrase search)

## Frontend Integration Notes

**For Phase 9 (Web Client) implementation:**

1. **Tiptap Editor Setup:**
   - Import from `@tiptap/react` and `@tiptap/starter-kit`
   - Use `EditorContent` component for rendering
   - Configure with `immediatelyRender: false` for SSR compatibility

2. **Auto-Save Implementation:**
   - Use the `useDebounce` hook pattern
   - 500ms debounce per NOTE-08 requirement
   - Show visual "Saving..." indicator

3. **Search UI:**
   - Debounce search input (300ms recommended)
   - Display ranked results with highlighted matches
   - Show "No results found" when search_vector returns empty

4. **Mood Tracking:**
   - Radio button or emoji selector for 1-5 scale
   - Line chart for mood trends (use Recharts in Phase 9)
   - Emoji grid for quick scan of recent entries

5. **Folder/Tag Organization:**
   - Single-level folder hierarchy (no nesting)
   - Tag pills with color coding (Phase 9 will define colors)
   - Filter notes by folder or tag
