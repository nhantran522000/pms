<context>
The original request was to:
1. Use @nx/js:tsc executor instead of SWC for feature-auth (preserves decorators better)
2. Investigate SWC configuration for decorator metadata
3. Run /gsd:pause-work

Discovery: The .swcrc files already have `decoratorMetadata: true`. The Nx TypeScript plugin is designed to use SWC for fast builds and TSC for type checking (hybrid approach). This is actually the optimal setup - SWC handles compilation with decorator support, TSC handles validation.

The issue with skipTypeCheck: true was that it was hiding real type errors (Zod v4, error handling, missing types). We've fixed those, so the SWC + .swcrc setup should work correctly now.
</context>
